# 验收标准

## 阶段验收项

1. 是否符合用户批准的阶段方案。
2. 是否只改动了必要范围。
3. 是否完成自测并提供结果。
4. 如涉及界面、交互或视觉调整，是否说明影响范围并完成验证。
5. 项目是否能按约定方式运行或查看。
6. 是否记录了已知问题和未覆盖风险。
7. 当天开发日志是否已更新。
8. 如直接完成实现工作，是否说明了修改范围、自测方式、自测结果和遗留风险。

## 计算网专项验收

涉及 B300/POD/Leaf/Spine 计算规则或拓扑展示时，必须验收以下内容：

1. B300=1-4：双 Spine 直连，Leaf=0，Spine=2；Spine 横联备注按 `B300 数量 * 8 * 400G` 显示。
2. B300=5-128：虚拟双平面；POD=`ceil(B300/32)`；Leaf 按 POD 分段汇总。
3. B300=129-224：物理双平面；POD=`ceil(B300/32)`；Leaf=`POD*16`。
4. 任一 POD 的 Leaf 数不得超过 16，P1/P2 单平面不得超过 8。
5. 清单 Leaf 总数、拓扑 POD 内 Leaf 展示、底部摘要 Leaf 数必须一致。
6. `计算网 LEAF 交换机` 的计算依据必须直接说明 Leaf 计算口径。

## 必测用例

- B300=1：Leaf=0，Spine=2，Spine 横联 `8*400G`。
- B300=4：Leaf=0，Spine=2，Spine 横联 `32*400G`。
- B300=5：POD=1，Leaf=4，Spine=2。
- B300=28：POD=1，Leaf=16，Spine=8。
- B300=33：POD=2，Leaf=18，Spine=16。
- B300=51：POD=2，Leaf=32，Spine=16；拓扑不允许出现单 POD 20 台 Leaf。
- B300=65：POD=3，Leaf=34，Spine=32。
- B300=81：POD=3，Leaf=48，Spine=32。
- B300=97：POD=4，Leaf=50，Spine=32。
- B300=128：POD=4，Leaf=64，Spine=32。
- B300=129：POD=5，Leaf=80，Spine=64。
- B300=138：POD=5，Leaf=80，Spine=64。
- B300=225：提示三层后续支持，不生成错误二层结果。

## 用户最终确认

阶段验收通过不等于阶段结束。只有用户明确批准验收通过，当前阶段才算暂时结束。

## 验收不通过处理

如果验收不通过，应说明问题、影响范围和下一步修订任务，整改后重新验证。

涉及功能或样式修改时，必须完成必要自测并记录风险。
