# 验收标准

## 当前负责人验收项

1. 是否符合用户批准的阶段方案。
2. 是否只改动了必要范围。
3. 是否完成自测并提供结果。
4. 如涉及界面、交互或视觉调整，是否说明影响范围并完成验证。
5. 项目是否能按约定方式运行或查看。
6. 是否记录了已知问题和未覆盖风险。
7. 当天开发日志是否已更新。
8. 当前负责人如亲自完成实现工作，是否说明了修改范围、自测方式、自测结果和遗留风险。

## 用户最终确认

当前负责人验收通过不等于阶段结束。只有用户明确批准验收通过，当前阶段才算暂时结束。

## 验收不通过处理

如果验收不通过，当前负责人应说明问题、影响范围和下一步修订任务，并直接整改后重新验证。

当前最高授权生效期间，当前负责人可以直接修改功能或样式；修改后必须完成必要自测并记录风险。
