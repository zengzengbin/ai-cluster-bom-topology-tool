# 设计规范

## 基本原则

1. 网页工具应以实用为主，先保证信息清晰、操作顺畅，再做视觉美化。
2. 页面布局要适合反复使用，避免只像展示页而不便操作。
3. 视觉风格可以逐步提升，但不能影响功能识别和数据准确性。
4. 所有控件、状态和提示应服务于真实工作流程。
5. 设计修改需要能被当前负责人和用户验收。

## 视觉方向

- 整体方向为“科技 + 时尚 + 克制高级感”，界面应清爽、安静、可长期使用。
- 主色以白色、银灰、浅石墨和深石墨文字为主，少量使用科技蓝或青色强调当前状态、主要操作和拓扑关键路径。
- 不使用大面积炫光、复杂渐变、过重阴影或影响数据阅读的背景图。
- 本项目是实用网页工具，不做品牌官网或营销页。任何视觉处理都必须服从参数输入、清单结果、拓扑画布、错误提示和总览信息的清晰可用。

## 页面结构

页面采用单页工作台结构，按用户操作顺序从上到下组织：

1. 顶部状态与参数输入区。
2. 结果总览区。
3. 四网清单区。
4. 四网独立拓扑区。
5. 统一大拓扑区。

## 表格规范

1. 四网清单表格应优先保证字段对齐、列宽稳定和横向扫描效率。
2. 表头固定使用浅灰背景或清晰分割线，字段名短而准确。
3. 关键字段如产品名称、产品型号、数量、计算依据必须保持可复制文本。
4. 数量字段使用强调色，但不能影响其他列阅读。
5. 计算依据必须直接说明当前行数量的来源，不能让用户看完仍无法判断该说明对应哪一列。
6. 对 `计算网 LEAF 交换机`，计算依据必须以 Leaf 为主语，推荐文案：`Leaf 按 POD 分段汇总：满编 POD=16 台 Leaf；唯一/最后 POD 按剩余 B300 的 CX8*2/32 计算后向上取 2 的幂；单 POD≤16，单平面≤8。`

## 拓扑区通用规范

1. 拓扑图应以“可读”为第一优先级，节点间距、连线方向和网络分组要清楚。
2. 每张拓扑必须先画区域，再画设备，再画链路；区域名称、设备层级和链路类型都应可见。
3. 链路只画代表性连接，数量通过标注表达；当设备数量较多时使用省略号和“同模式扩展”说明。
4. 拓扑中的数量、POD 数、Leaf 数和 Spine 数必须与清单一致。
5. 拓扑画布允许容器内部横向滚动，但不应造成整页横向溢出。
6. 每张拓扑都需要局部说明，说明颜色、线型、链路速率和省略号含义。

## 计算网独立拓扑规范

计算网拓扑优先采用 `Spine 区 + Leaf 区 + POD 区 + P1/P2 平面 + 服务器 8 张 CX8 网卡条` 的表达方式。

### B300=1-4：双 Spine 直连

- 不展示 Leaf 层。
- 展示两台 Spine 与 1-4 台代表服务器。
- Spine 横联备注按 `B300 数量 * 8 * 400G` 显示，例如 `8*400G`、`32*400G`。
- 服务器到双 Spine 的蓝色连线可画代表链路，底部说明需写明“图中仅画代表链路”。

### B300=5-128：虚拟双平面

- 需要展示 Spine 区、Leaf 区和 POD 区。
- POD 数按 `B300 / 32` 向上取整。
- Leaf 展示必须按 POD 分段，不允许把全网 Leaf 平均或轮询追加到已满 POD。
- 满编 POD 展示 16 台 Leaf，P1/P2 各 8 台。
- 唯一 POD 或最后 POD 按本 POD 实际 B300 数量计算 Leaf，结果需要向上取到 2 的幂，且单 POD 不超过 16。
- 当 B300=51 时，拓扑应表达 POD1=16、POD2=16，不允许出现 POD1=20。
- 当 B300=65 时，拓扑应表达 POD1=16、POD2=16、POD3=2，总 Leaf=34。

### B300=129-224：物理双平面

- POD 数按 `B300 / 32` 向上取整。
- 每个 POD 固定 16 台 Leaf，P1/P2 各 8 台。
- 物理双平面需要保留 Spine 逃生链路视觉标注。
- 末组不足 32 台 B300 时，仍按完整 POD 展示 16 台 Leaf。

### 压缩显示策略

- 当 POD 数量较少时，尽量完整展示所有 POD。
- 当画布空间不足时，可展示 POD1、POD2、PODn 代表框，并用省略号表达中间 POD。
- 即使压缩显示，底部摘要也必须显示真实 `B300 / POD / Leaf / Spine` 数值。

## 统一大拓扑规范

统一大拓扑只做总览，不直接复用四张子拓扑画布。

- 服务器源区表达 B300、全闪、混闪、管理服务器。
- 计算网只连接 B300 计算网卡。
- 存储网连接 B300 与全闪的 400G 存储网卡。
- 带内管理网连接四类服务器的带内管理口。
- 带外管理网连接四类服务器 BMC 以及网络/安全设备管理口。

## 设计调整汇报要求

每次汇报必须包含：

- 设计范围。
- 调整内容。
- 涉及文件。
- 验证方式。
- 遗留风险或需要用户确认的问题。
