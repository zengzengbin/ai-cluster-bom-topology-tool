# 开发需求

## 当前目标

建设一个本地网页工具，用于售前根据智算项目服务器数量和服务器网卡配置，自动生成四张网络的清单和网页拓扑：

- 计算网
- 存储网
- 带内管理网
- 带外管理网

第一阶段只做四网 MVP，先覆盖需求文档中的当前二层组网规则，不做三层计算网、导出、历史保存、账号系统或数据库。

## 用户输入边界

用户在工具中只需要填写两类信息：

1. 服务器数量：
   - B300 GPU 服务器数量
   - 全闪存服务器数量
   - 混闪存服务器数量
   - 管理服务器数量
2. 各类服务器的网卡配置：
   - GPU 服务器存储网卡配置
   - 全闪服务器存储网卡配置

工具内部负责把网卡配置换算成存储网 400G 接入口、带内 25G 接入口和带外管理口等推导数据。第一阶段不要求用户直接填写接入口数、端口数或公式结果。

## 第一阶段功能范围

1. 页面以本地网页服务形式运行。
2. 输入参数后实时生成四网清单。
3. 每条清单展示：网络、序号、产品名称、产品型号、产品描述、数量、计算说明。
4. 每张网展示一个网页内拓扑图。
5. 额外展示一个统一大拓扑，表达四张网络与服务器、交换、安全和管理设备之间的整体关系。
6. 计算网只支持 B300 二层组网；B300 数量超过 224 台时提示三层后续支持，不生成错误二层结果。
7. 计算网清单与计算网拓扑必须保持数量同源：Leaf 总数、POD 数、每 POD Leaf 数和 Spine 数不得互相矛盾。

## 计算网关键需求

- B300=1-4：双 Spine 直连，不设置 Leaf；Spine 横联备注按 `B300 数量 * 8 * 400G` 显示。
- B300=5-128：虚拟双平面；POD 数按 `B300 / 32` 向上取整；Leaf 按 POD 分段汇总，满编 POD 固定 16 台 Leaf，唯一/最后 POD 按剩余 B300 的 `CX8*2/32` 计算后向上取 2 的幂；单 POD 不超过 16，单平面不超过 8。
- B300=129-224：物理双平面；POD 数按 `B300 / 32` 向上取整；Leaf 仍按 `POD 数 * 16`。

## 验收用例

- B300=4：计算网为双 Spine 直连，Leaf=0，Spine=2，Spine 横联显示 `32*400G`。
- B300=5：计算网为虚拟双平面，POD=1，Leaf=4，Spine=2。
- B300=28：计算网为虚拟双平面，POD=1，Leaf=16，Spine=8。
- B300=33：计算网为虚拟双平面，POD=2，Leaf=18，Spine=16。
- B300=51：计算网为虚拟双平面，POD=2，Leaf=32，Spine=16；拓扑中任何单 POD Leaf 不得超过 16。
- B300=65：计算网为虚拟双平面，POD=3，Leaf=34，Spine=32。
- B300=97：计算网为虚拟双平面，POD=4，Leaf=50，Spine=32。
- B300=128：计算网为虚拟双平面，POD=4，Leaf=64，Spine=32。
- B300=138：计算网为物理双平面，POD=5，Leaf=80，Spine=64。
- GPU=128，全闪=8，GPU 与全闪均为每台 2*400G：存储接入口=272，Leaf=10，Spine=8。
- GPU=128，全闪=8，混闪=16，管理=8：带内 25G 接口=320，带内 Leaf=8。
- B300>224：页面提示三层后续支持，不生成错误二层结果。

## 已确认协作需求

1. 当前会话负责人是项目唯一执行入口，负责需求理解、方案规划、实现、验收和结果汇总。
2. 用户直接向当前负责人提出需求，当前负责人直接完成实现或文档更新。
3. 每次实现或设计调整完成后，必须自测并保留可核查结果。
4. 用户批准验收通过后，当前阶段才算结束。
5. 项目推进不能一次做太多，应保持稳定、安全、有效。

## 第一阶段不做事项

- 不做三层计算网。
- 不做清单或拓扑导出。
- 不保存历史项目数据。
- 不接入后端、数据库、账号或云端服务。
- 不追求专业制图软件级拓扑，只保证结构和数量关系可读。
