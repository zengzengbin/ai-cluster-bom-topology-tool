# 计算网规则

说明：本文件沉淀计算网规则，作为当前实现与验收参考。

## 通用约束

- 单台 B300 固定 8 张 CX8。
- 一个 POD 最多 32 台 B300。
- 一个 POD 最多 16 台 Leaf，即 P1/P2 两个平面各最多 8 台 Leaf。
- 计算网清单和计算网拓扑必须使用同一套派生结果，不能出现清单 Leaf 数量与拓扑 Leaf 分布不一致。
- 拓扑允许只画代表链路，但数量说明必须与清单一致。

## B300 数量 1-4

- 组网：双 Spine 直连，不设置 Leaf 层。
- 拓扑：画两台 Spine，按实际 B300 数量画 1-4 台服务器；每台服务器上联 16 根链路，平均分布到两台 Spine；两台 Spine 之间有横联链路。拓扑中可画代表链路并配备注，不要求把全部链路画满。
- 拓扑备注：Spine 横联按 `B300 数量 * 8 * 400G` 表达，例如 B300=1 写 `8*400G`，B300=4 写 `32*400G`。
- 清单：
  - 计算网 SPINE 交换机：固定 2 台。
  - SPINE 互联光模块：`400G-Q112-DR4-L`，数量 `CX8 网卡数 * 2`。
  - SPINE 间互联光纤-50 米：`MPO-MPO-SM-10M(APC)`，数量 `SPINE 互联光模块 / 2`。
  - SPINE 下联光模块：`400G-Q112-VR4-MM850`，数量 `CX8 网卡数 * 2`。
  - 服务器侧 800G 光模块：`800G-OSFP-RHS-2VR4-MM850`，数量 `CX8 网卡数`。
  - 交换机-服务器互联光纤：`MPO-MPO-OM4-30M(APC)`，数量 `CX8 网卡数 * 2`。

## B300 数量 5-128

- 组网：虚拟双平面。
- POD 数量：`B300 服务器数量 / 32`，向上取整。
- Leaf 数量：按 POD 分段计算后汇总。
  - 满编 POD：每 32 台 B300 固定 16 台 Leaf，即 P1/P2 各 8 台。
  - 唯一 POD 或最后一个不足 32 台的 POD：按本 POD 实际 B300 数量计算，公式为 `本 POD B300 数量 * 8 * 2 / 32`，先向上取偶数，再向上取 2 的幂。
  - 单 POD 结果不能超过 16；如计算结果超过 16，说明 POD 分段或规则实现有误。
- Spine 数量：`Leaf 数量 / 2`，结果向上取 2 的幂。
- 清单计算依据推荐文案：`Leaf 按 POD 分段汇总：满编 POD=16 台 Leaf；唯一/最后 POD 按剩余 B300 的 CX8*2/32 计算后向上取 2 的幂；单 POD≤16，单平面≤8。`
- 拓扑：按虚拟双平面绘制；每个 POD 的 Leaf 展示必须使用本 POD 计算结果，不允许把全局多余 Leaf 追加到已满 16 台的 POD。

## B300 数量 129-224

- 组网：物理双平面。
- POD 数量：`B300 服务器数量 / 32`，向上取整。
- Leaf 数量：`POD 数 * 16`。
- Spine 数量：`Leaf 数量 / 2`，结果向上取 2 的幂。
- 拓扑：按物理双平面绘制；每个 POD 固定 16 台 Leaf，P1/P2 各 8 台。
- 额外清单：
  - SPINE 逃生链路 LPO 光模块：`400G-Q112-DR4-L`，数量 `Spine 数量 * 8`。
  - SPINE 逃生链路互联光纤-10 米：`MPO-MPO-SM-10M(APC)`，数量 `SPINE 逃生链路 LPO 光模块数量 / 2`。
